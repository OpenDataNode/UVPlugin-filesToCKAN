L-FilesToCkan loader can be used to load files to CKAN as CKAN resources.
Name of CKAN resource is equal to file name.

In order to work properly, these two parameters must be set in backend config.properties:

  • org.opendatanode.CKAN.api.url - URL of CKAN API
  • org.opendatanode.CKAN.secret.token - authentication token for target CKAN

Optional HTTP header parameters to be sent to CKAN can be defined in backend config.properties as:
org.opendatanode.CKAN.http.header.[key]


Also, prior to running this DPU, an association between pipeline and CKAN dataset must exist, otherwise DPU fails

List of options:

  • CKAN resource name - Name of resource to be created in CKAN. It should not change between pipeline runs as this name is used as key to CKAN.
  • Use file name as CKAN resource name - (default false) If selected, file name is used as CKAN resource name. When multiple input files, this option must be selected. Otherwise DPU fails
  • Overwrite existing resources - (default false) if true and resource with the same name already exists in associated dataset, it is overwritten.
