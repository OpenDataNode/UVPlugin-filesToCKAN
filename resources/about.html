L-FilesToCkan loader can be used to load files to CKAN as CKAN resources.
Name of CKAN resource is equal to file name.

In order to work properly, these two parameters must be set in backend config.properties:

  • dpu.l-filesToCkan.catalog.api.url - URL of CKAN API
  • dpu.l-filesToCkan.secret.token - authentication token for target CKAN

Also, prior to running this DPU, an association between pipeline and CKAN dataset must exist, otherwise DPU fails

List of options:

  • Overwrite existing resources - (default false) if true and resource with the same name already exists in associated dataset, it is overwritten. By default nothing happens if already exists, just warning message
